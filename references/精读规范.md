# 命理经典精读规范

## 目标
不是"总结大意"，而是**吃透每一章的所有规则**，让AI断命时能精准引用。

## 每章必须包含（缺一不可）

### 1. 原文金句
- 直接引用文中原话（文言文），让人知道这条规则出自哪里
- 格式：`> 「原文内容」`

### 2. 核心规则（逐条拆解）
- 每条可操作的判断规则单独列出
- 格式：**规则名** → 具体条件 → 结论
- 每章至少5条规则，不许凑数
- 必须包含"何时用"和"何时不用"的边界条件

### 3. 格局成败分析
- 成格条件：什么情况下这条规则生效
- 破格条件：什么情况下会被破坏
- 救应方法：破了之后有没有补救

### 4. 案例拆解
- 书中命例：完整四柱 + 推理过程 + 结论
- 如果原文无完整命例，用条件例说明

### 5. 例外和易错点
- 常见误判场景
- 看似对其实错的口诀
- 与其他规则冲突时的处理

### 6. 章际关联
- 本章承接哪章、延伸到哪章
- 与其他三本书的交叉验证点

## 精读批次策略

### 核心原则：一次读太多章会导致每章只有5条bullet
- 渊海子平：2467行 → 分3批 → 每批约800行
- 滴天髓：11590行 → 分4批 → 每批约2800行
- 三命通会：~400KB → 按卷分批（12卷，每卷1-2批）
- 子平真诠：704行 → 分5批 → 每批约140行

### 并行执行方法
1. 每批用独立的 `codex exec --full-auto` 进程
2. 每批写独立文件（如 `渊海子平_批1_基础十神.md`）
3. 全部完成后合并成一个精读文件
4. 清理所有批文件
5. 更新SKILL.md索引

### 合并+清理流程（2026-05-07验证）
```bash
# 1. 合并
cat 书名_批1_*.md 书名_批2_*.md ... > knowledge/书名精读.md

# 2. 验证
wc -l knowledge/书名精读.md  # 应接近各批行数之和
ls -la knowledge/书名精读.md  # 文件大小应>50KB

# 3. 清理
rm 书名_批*.md

# 4. 更新SKILL.md
# 在"精读笔记"章节添加新书的条目
```
